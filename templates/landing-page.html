Designing with Code

"Build real things to test real behavior."
- Clayton Farr

Build Interactive Prototypes

When designers code, their designs create the illusion of an actual working product. That code should be quick, dirty, and prototype-worthy. Not production-ready.

Prototype on the web

Easily send a link of the latest prototype to users. They can view it on desktop or mobile without any special downloads.

Reusable components

When building a new component for production our interactive prototypes are referenced. Once it's in production, it becomes documented in the production tab, so that it can be reused.

Become a Design Wizard

Several things can get lost when delivering a static mockup to a developer. Motion, animation, interactivity, and polish are never lost when designing with code.

Speed up Development

Although prototypes are encouraged to be quick and dirty, engineers hit the ground running with reusable HTML and CSS patterns.

Living Design Guide

Both engineers and designers can reference the online interactive style guide. Anyone in the organization can access and understand the product design guidelines.

Software Engineering Process

Designers

Developers

WHITEBOARD & SKETCH

Designers sketch, mockup, and design new concepts referencing our existing style guide inside particle.

WHITEBOARD & SKETCH

Engineers participate early in brainstorming sessions and provide valuable feedback, ideas, and insight.

PROTOTYPE

Designers build interactive prototypes using particle components. Designers code what is necessary to get valuable feedback. Often images are used within prototype to fake it.

USER TESTING

New designs are given to users, and tested thoroughly. Through directed discovery designers identify new problems & opportunities.

REFINE PROTOTYPE

Designers finalize design decisions based on feedback from users, product managers, and engineers.

VALIDATE PROTOTYPE

Engineers clarify anything missing in the prototype, compare to existing product, and offer feedback.

REFERENCE PROTOTYPE

Once the design has stablized, engineers are given an interactive prototype of the latest design.

VERIFY PRODUCTION CODE

Designers are involved during production and part of the code review. Verifying that production is in line with the prototype and current design guidelines.

BUILD PRODUCTION

Reusing HTML and CSS where it makes sense, engineers build a scalable, production-ready version of the prototype.

UPDATE STYLE GUIDE

Newly built components are added to particle design to be reused in the future.
